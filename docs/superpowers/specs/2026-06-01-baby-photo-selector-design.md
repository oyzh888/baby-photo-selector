# 宝宝精选 Baby Photo Selector — 产品设计文档

**日期：** 2026-06-01
**版本：** V1.0（MVP）
**目标：** 尽快上架 iOS App Store

---

## 一、产品概述

### 产品定位
一款帮助新手父母从海量手机照片中自动筛选出宝宝最佳照片的 iOS 工具 App。用户授权相册后，App 自动扫描最近 1000 张照片，通过本地 AI 识别宝宝照片并评分，挑出最好看的 20~50 张，支持手动微调后一键保存为系统相册中的「宝宝精选」文件夹。

### 商业模式
- **免费下载**，面向所有新手父母
- V1 不含广告，专注用户口碑
- 后续版本可叠加：订阅解锁高级功能（扫描更多张、自定义评分权重等）

### 核心价值主张
- 全本地处理，**照片不离开手机**，隐私安全
- 无需手动翻看，**一键自动筛选**
- 误判可手动纠正，**结果可控**

## 二、技术选型

### 开发框架
- **Expo（Bare Workflow）**：快速开发，支持 OTA 更新，后续按需迁移 SwiftUI
- 目标平台：iOS 优先，代码结构预留 Android 扩展空间

### 核心依赖
| 功能 | 库 | 说明 |
|------|-----|------|
| 相册访问 | `expo-media-library` | 封装 PhotoKit，支持全部/选中两种权限模式 |
| 人脸检测 + 年龄判断 | `@react-native-ml-kit/face-detection` | Google ML Kit，本地运行，iOS/Android 通用 |
| 清晰度/曝光评分 | JS 自研（图像统计） | 基于缩略图像素分析，无原生依赖 |
| 重复照片去重 | JS 自研（时间窗口聚类） | 60 秒内连拍归为同组，取最高分 |
| 保存相册 | `expo-media-library` | 写入 PHAssetCollection「宝宝精选」 |

### 性能策略
- 全程使用**缩略图**（thumbnail）处理，不加载原图
- 扫描在**后台线程**运行，主线程保持 UI 响应
- 支持锁屏后台继续扫描（BackgroundFetch）

## 四、AI 评分算法（V1）

### 筛选流水线
```
读取最近 1000 张照片（缩略图）
  └─> Step 1：ML Kit 人脸检测 → 无人脸的直接过滤
        └─> Step 2：年龄段判断 → 非婴幼儿脸过滤（婴儿概率 < 0.5 过滤）
              └─> Step 3：综合评分 → 计算每张得分
                    └─> Step 4：连拍去重 → 同组保留最高分
                          └─> Step 5：取 Top 20~50 张输出
```

### 综合评分权重（V1）
| 维度 | 权重 | 实现方式 |
|------|------|---------|
| 人脸清晰度 | 35% | 缩略图拉普拉斯方差，越高越清晰 |
| 婴儿概率 | 30% | ML Kit 返回的年龄置信度评分 |
| 曝光正常 | 20% | 像素亮度均值，过暗/过曝扣分 |
| 人脸构图 | 15% | 人脸 bounding box 面积 / 图片面积 |

### 输出控制
- 默认输出 **Top 30 张**，用户可在设置中调整（20 / 30 / 50）
- 同一天内最多输出 **5 张**（避免某天拍了很多导致其他天没有）

## 五、页面详细设计

### 页面1：首页（Home）
**内容：**
- App Logo + 标语："帮你从海量照片里找出宝宝最美的瞬间"
- 主按钮：「开始扫描」
- 次要入口：上次扫描时间 + 「查看上次结果」（有历史结果时显示）

**状态：**
- 首次打开：只显示「开始扫描」
- 有历史结果：显示上次扫描时间和查看入口

---

### 页面2：授权引导页（Permission）
**内容：**
- 插画（宝宝照片相关）
- 说明文字：为什么需要相册权限（强调本地处理、隐私安全）
- 主按钮：「允许访问全部照片」→ 请求系统权限 → 权限通过后进入扫描
- 次要按钮：「只选择部分照片」→ 调起系统照片选择器 → 扫描选中的
- 拒绝处理：显示「去设置中开启」按钮 + 说明引导

**权限类型支持：**
- `PHAuthorizationStatusAuthorized`（全部访问）：自动扫描最近 1000 张
- `PHAuthorizationStatusLimited`（有限访问）：扫描用户选中的照片

---

### 页面3：扫描进度页（Scanning）
**内容：**
- 动态进度条
- 实时文字："正在分析第 342 张…"
- 预估剩余时间
- 可最小化到后台，锁屏不中断

**技术细节：**
- 每批处理 20 张，处理完一批更新进度 UI
- 进度状态持久化，App 被杀后重启可继续（或从头再扫）

---

### 页面4：精选结果页（Results）— 核心页面
**布局：**
- 顶部标题：「为你找到 XX 张宝宝精选」+ 副标题说明
- 3 列瀑布流网格展示
- 默认全选（照片右上角蓝色对勾）
- 底部固定：「保存精选（XX 张）」按钮

**交互：**
| 操作 | 行为 |
|------|------|
| 单张点击 | 全屏预览（支持左右滑动切换） |
| 单张长按 | 弹出「移除」选项 |
| 右上角切换 Tab | 「AI 精选」/ 「全部照片」 |
| 「全部照片」Tab | 展示原始 1000 张，可点击添加到精选 |
| 顶部「重新扫描」| 清空结果，重新开始 |

**全屏预览页：**
- 显示该照片的 AI 评分详情（清晰度、表情、曝光各项得分）
- 「移除」/ 「保留」按钮

---

### 页面5：完成页（Done）
**内容：**
- 成功提示：「已保存 XX 张到「宝宝精选」相册 🎉」
- 「在系统相册查看」→ 跳转系统相册 App
- 「分享给家人」→ 系统分享面板（可分享截图或相册链接）
- 「重新扫描」→ 返回首页

## 六、数据模型

### 本地存储（AsyncStorage）
```typescript
// 上次扫描结果
interface ScanResult {
  scannedAt: number;          // 时间戳
  totalScanned: number;       // 扫描总数
  candidates: PhotoCandidate[]; // AI 候选照片
  selected: string[];         // 用户最终选中的 localIdentifier 列表
}

// 单张照片候选
interface PhotoCandidate {
  localIdentifier: string;    // PhotoKit 唯一 ID
  uri: string;                // 缩略图 URI
  creationTime: number;       // 拍摄时间
  scores: {
    sharpness: number;        // 清晰度 0~1
    babyProbability: number;  // 婴儿概率 0~1
    exposure: number;         // 曝光评分 0~1
    composition: number;      // 构图评分 0~1
    total: number;            // 综合得分 0~1
  };
}
```

## 七、用户流程总览

```
启动 App
  └─> 首次打开 → 授权引导页
        ├─> 全部授权 → 扫描进度页（自动扫描最近1000张）
        └─> 部分授权 → 系统选择器 → 扫描进度页
  └─> 再次打开（有历史）→ 首页（可查看上次结果或重新扫描）

扫描完成 → 精选结果页
  ├─> 移除误判（长按 → 移除）
  ├─> 补充漏选（切换「全部照片」Tab → 点击添加）
  └─> 确认 → 「保存精选」→ 完成页

完成页
  ├─> 查看系统相册
  ├─> 分享给家人
  └─> 重新扫描
```

## 八、MVP 范围（V1 不做）

以下功能**V1 明确不做**，避免范围蔓延：

- 视频筛选
- 多宝宝区分（家里有多个小孩）
- 自定义评分权重（用户调参）
- 云端备份 / iCloud 同步
- Android 版本
- 广告 / 付费墙
- 社交分享到微信/微博
